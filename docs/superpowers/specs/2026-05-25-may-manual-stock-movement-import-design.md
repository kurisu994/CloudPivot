# 2026 年 5 月自由出入库录入脚本设计

## 目标

将 `scripts/酉昌进销存明细.xlsx` 的 `5月` 工作表转换为可在 CloudPivot PostgreSQL 数据库执行的录入脚本。脚本仅处理 2026 年 5 月数据，并以系统现有“期初库存导入 + 自由出入库”业务语义落账，不创建采购单、销售单、应收或应付记录。

## 已确认口径

- 数据日期按 `2026-05-01` 至 `2026-05-31` 解释；源表实际仅在 `2026-05-01` 至 `2026-05-22` 存在非零日发生额。
- 先导入 `期初数据` 中物料可匹配且数量大于零的库存，再处理每天的 `入` / `出` 列；对参与日发生但品号为空的两条记录，在导入脚本内先创建独立基础物料。
- 日发生额仅以每日列为准；汇总列用于核对，不用于补造缺少日期的交易。
- 同一物料同一天先处理入库，再处理出库。
- 日常入库与出库按自由出入库记录为 `other_in` / `other_out`，生成 `FM-YYYYMMDD-XXX` 关联单号。
- 若某笔出库会导致库存小于零，在该笔出库前先生成一笔同日 `other_in`，备注为源表负库存补足，数量仅补足本次出库所需缺口。
- Excel 未提供单位成本。期初、原始日入库及负库存补足入库的 `unit_cost` 均按 `USD 0` 录入。

## SQL 脚本结构

### 前置校验

脚本在写入前验证以下条件，任一不满足则中止事务：

1. `scripts/seed_materials_pg.sql` 所覆盖的本次交易物料已存在且处于启用状态；脚本补建的两条无品号日发生物料可在同一事务内初始化。
2. `default_warehouses` 中存在启用的 `raw` 默认仓库；本次 Excel 真实物料均按原材料仓处理。
3. 匹配到的物料 `lot_tracking_mode = 'none'`。基础 SQL 当前采用默认值；若执行前人工启用批次追踪，脚本拒绝绕过批次逻辑。
4. `users` 中存在启用的 `admin` 账号，用于脚本落下的操作人快照。
5. 数据库尚无带有本次导入标记的库存流水，防止脚本重复执行。

### 期初库存

期初数据复现现有 `import_initial_inventory` 的结果：

- 对 354 行可录入且期初数量大于零的数据写入 `inventory`。
- 写入 `inventory_transactions`，类型为 `other_in`，`source_type = 'initial_inventory'`，`related_order_no = 'INIT'`。
- 所有期初单位成本为 `0`，不创建批次记录。
- 在期初完成后写一条操作日志，说明本次导入的行数和来源文件。

### 每日自由出入库

脚本按日期、物料顺序处理每日发生额：

- 原始日入库按 `other_in` 入账，并按现有移动平均规则更新 `inventory`；成本为 `0`。
- 原始日出库按 `other_out` 入账，扣减当前库存并继承当前平均成本。
- 每一笔原始日交易按应用自由出入库的审计语义写入操作日志。
- 单号按当日数据库中已有 `FM-YYYYMMDD-XXX` 最大序号继续递增，避免覆盖同日已有手工记录。

### 负库存补足

处理出库前读取当前库存；若当前数量不足：

- 差额作为同日 `other_in` 自由入库，成本为 `0`。
- 备注和操作日志明确标注 `源表负库存补足`、目标物料以及被补足的原始出库日期。
- 补足入库也生成独立 `FM-YYYYMMDD-XXX` 单号，之后再写原始出库单号。

### 幂等与事务

- 整个脚本在单个事务内执行；任一前置校验或写入失败均不留下部分导入结果。
- 所有本次写入的流水备注带固定来源标记 `[酉昌2026-05导入]`。
- 执行前若发现该来源标记已存在，直接失败；需要重导时应先由人工审核并清理已有导入数据，不在脚本中自动删除历史记录。

## 与应用逻辑的一致性

该方案是数据导入 SQL，不会实际调用 Tauri IPC。脚本复现本次数据涉及的业务结果：

- 更新 `inventory.quantity`、`avg_cost`、`last_in_date`、`last_out_date`。
- 生成 `inventory_transactions` 的 `other_in` / `other_out` 流水与自由出入库单号。
- 写入与自由出入库、期初导入相对应的 `operation_logs`。
- 不修改采购、销售、财务、预留或批次数据。

由于基础物料未开启批次追踪，脚本不涉及 `inventory_lots`。前置校验确保后续若业务配置改变，不会静默偏离应用的 FIFO 批次规则。

## 验收

生成脚本后执行静态数据核对：

- 输入原始日发生为 441 笔，其中包含两条无品号日发生物料的 3 笔交易。
- 生成期初流水为 354 笔。
- 生成日常自由出入库流水为 463 笔，其中补足入库为 22 笔、数量合计为 1200。
- 日列与汇总不一致、未匹配物料、负库存补足记录均出现在异常报告中。
- SQL 仅使用固定生成数据和数据库变量绑定/关系查询，不拼接来自运行时用户输入的可执行 SQL。

如具备可安全测试的 PostgreSQL 数据库，额外以事务执行并回滚方式验证前置校验、记录数量与最终非负库存；若无测试数据库，则仅报告静态校验结论，不将未执行结果表述为已落库。
