# 批量出入库单需求与设计规格

| 项目 | 内容 |
| --- | --- |
| 产品 | 云枢 (CloudPivot IMS) |
| 日期 | 2026-05-27 |
| 状态 | 业务方案已确认，待实施计划 |
| 替代能力 | 当前「自由出入库」单物料即时过账页面 |
| 相关模块 | 库存、库存流水、智能补货、库存报表、操作日志 |

## 1. 背景与结论

当前「自由出入库」每次只能选择一个物料并立即过账。实际作业中，用户经常需要对同一仓库中的多种物料同时进行入库或出库，逐笔重复填写方向、仓库、日期和原因效率过低。

经需求确认，典型操作具备以下特征：

- 一次操作通常为同一方向、同一仓库。
- 明细规模通常不超过 100 行。
- 用户需要先保存草稿，核对后再过账。
- 用户逐行选择物料并填写数量即可，首版不需要 Excel 导入。
- 用户需要用可读的业务类型描述手工过账原因，例如「采购入库」「生产领料」「外借出库」。

因此，本需求值得实施，推荐将「自由出入库」升级为「批量出入库单」：以草稿单据承载最多 100 条明细，确认时在一个事务内完成整单过账。

## 2. 目标

### 2.1 用户目标

- 一次录入并过账同仓库、同方向的多种物料。
- 录入过程中可保存草稿，稍后继续核对或修改。
- 能按实际业务原因追溯手工库存变动。
- 借入、外借及其归还操作能够分类记录往来对象。

### 2.2 系统目标

- 批量过账必须原子执行，禁止出现部分明细已生效、部分失败的状态。
- 手工业务类型不得冒充正式采购单、生产工单等业务来源。
- 「生产领料」类型的手工出库可进入消耗分析和补货建议。
- 正式采购、应付、生产工单成本等统计口径不因手工类型展示名称而被污染。

## 3. 范围

### 3.1 首版包含

- 批量出入库单列表、草稿编辑和确认过账。
- 同一张单最多 100 条物料明细。
- 单头统一方向、仓库、业务类型、变动日期、往来对象和备注。
- 入库成本与批次信息录入。
- 出库按现有 FIFO 规则扣减可用批次。
- 重复物料自动合并规则。
- 高风险过账二次确认。
- 库存流水显示和筛选手工业务类型及来源单据。
- 「生产领料」手工出库纳入消耗分析与补货建议。

### 3.2 首版不包含

- 一张单中混合入库和出库。
- 一张单跨多个仓库。
- Excel 粘贴或文件导入。
- 审批流。
- 已确认单据撤销、反审核或冲销。
- 借入/外借的原单核销、未归还余额和自动匹配。
- 管理员自定义业务类型。
- 手工「采购入库」进入正式采购排行、供应商统计或应付账款。
- 手工「生产领料」关联生产工单、BOM 展算或工单成本核算。

## 4. 业务流程

```
批量出入库单列表
  -> 新建草稿
  -> 填写单头并添加物料明细
  -> 保存草稿
  -> 继续编辑或删除草稿
  -> 确认过账（二次确认条件触发时先确认风险）
  -> 整单更新库存、批次和库存流水
  -> 在流水及单据详情中追溯
```

### 4.1 状态

| 状态 | 含义 | 可执行操作 |
| --- | --- | --- |
| `draft` | 草稿，尚未影响库存 | 编辑、保存、删除、确认过账 |
| `confirmed` | 已确认并完成过账 | 查看详情、查看流水 |

首版不设置审核人流程。「确认过账」由当前有权限的用户主动执行，确认后不可编辑。

### 4.2 单号

- 单号格式沿用 `FM-YYYYMMDD-XXX`。
- 草稿首次保存时生成单号，使用户可从列表找回未完成操作。
- 确认产生的每条库存流水共用该单号作为关联单据号。

## 5. 单头规则

一张批量出入库单必须统一以下字段：

| 字段 | 规则 |
| --- | --- |
| 方向 | `in` 入库或 `out` 出库；一张单不得混合 |
| 业务类型 | 必选；可选项随方向变化 |
| 仓库 | 必选；所有明细在同一仓库过账 |
| 变动日期 | 必选；所有明细使用同一业务日期 |
| 往来对象 | 仅借料类类型在确认前必填 |
| 备注 | 「其他入库」「其他出库」在确认前必填，其他类型可选 |

### 5.1 草稿保存校验

草稿保存至少要求：

- 方向、业务类型、仓库和变动日期已填写。
- 至少存在一条已选择物料且数量大于零的明细。
- 明细数量不超过 100 条。

为支持中途保存，入库成本、借料往来对象和「其他」备注可在草稿阶段暂缺，但必须在确认过账前补齐。

## 6. 业务类型与统计口径

业务类型是批量手工单的原因分类，不改变其「手工过账」来源身份。界面显示名称可与正式单据流程相同，但数据中必须保留来源差异。

### 6.1 入库类型

| 内部编码 | 显示名称 | 往来对象 | 统计口径 |
| --- | --- | --- | --- |
| `manual_purchase_in` | 采购入库 | 不填写 | 计入库存收发存；不计入正式采购、供应商或应付统计 |
| `borrowed_material_in` | 借入入库 | 必填 | 计入库存，不计入采购 |
| `lent_material_return_in` | 外借归还入库 | 必填 | 计入库存，不计入采购 |
| `adjustment_in` | 调整入库 | 不填写 | 计入库存 |
| `other_in` | 其他入库 | 不填写 | 计入库存；备注必填 |

### 6.2 出库类型

| 内部编码 | 显示名称 | 往来对象 | 统计口径 |
| --- | --- | --- | --- |
| `manual_production_out` | 生产领料 | 不填写 | 计入消耗分析和补货建议；不关联工单 |
| `borrowed_material_return_out` | 借入归还出库 | 必填 | 不计入生产消耗 |
| `lent_material_out` | 外借出库 | 必填 | 不计入生产消耗 |
| `scrap_out` | 报废出库 | 不填写 | 不计入生产消耗 |
| `sample_out` | 样品发放 | 不填写 | 不计入生产消耗 |
| `adjustment_out` | 调整出库 | 不填写 | 不计入生产消耗 |
| `other_out` | 其他出库 | 不填写 | 不计入生产消耗；备注必填 |

### 6.3 显示相同名称时的来源区分

库存流水中可能同时存在正式流程和手工批量单产生的同名记录。列表和详情必须展示来源单据：

| 类型显示 | 来源显示示例 | 数据语义 |
| --- | --- | --- |
| 采购入库 | `采购单 PO-20260527-001` | 正式采购流程 |
| 采购入库 | `批量出入库单 FM-20260527-001` | 手工采购到货记录 |
| 生产领料 | `生产工单 MO-20260527-001` | 正式生产工单领料 |
| 生产领料 | `批量出入库单 FM-20260527-002` | 无工单的手工生产消耗 |

## 7. 明细录入与合并规则

### 7.1 录入方式

- 用户在快速添加区选择物料并填写数量，然后点击加入明细。
- 加入后的明细在表格内可继续修改物料和数量。
- 入库明细还可编辑单位成本、批次号和供应商批次。
- 对启用批次追踪的物料，批次号为空时沿用现有自动生成规则。
- 对未启用批次追踪的物料，不要求录入批次字段。

### 7.2 明细字段

| 字段 | 入库 | 出库 | 规则 |
| --- | --- | --- | --- |
| 物料 | 必填 | 必填 | 必须为启用状态 |
| 数量 | 必填 | 必填 | 大于零，使用物料基本单位 |
| 单位成本 (USD) | 确认前必填 | 不适用 | 不得小于零 |
| 批次号 | 可选 | 不适用 | 批次追踪物料为空则自动生成 |
| 供应商批次 | 可选 | 不适用 | 仅作为批次追溯信息 |

首版不增加行备注；整单原因由单头备注统一记录，以避免合并规则产生备注冲突。

### 7.3 自动合并

| 场景 | 行为 |
| --- | --- |
| 出库添加重复物料 | 按物料合并，数量相加 |
| 出库修改物料后与已有行重复 | 自动并入已有行并移除当前重复行 |
| 入库添加或修改为相同物料，且单位成本、批次号、供应商批次均一致 | 自动合并，数量相加 |
| 入库为相同物料，但单位成本或任一批次字段不同 | 保留多行，不合并 |

合并后界面提示用户，例如：「已合并至第 2 行，数量更新为 35」。

前端负责提供即时交互反馈，后端保存时仍需按同一规则规范化或拒绝不满足约束的数据，不能仅依赖前端保证一致性。

## 8. 过账与风控规则

### 8.1 确认前校验

确认过账前必须校验：

- 单据仍为 `draft` 状态。
- 单头必填字段及按业务类型要求的往来对象、备注完整。
- 明细非空且不超过 100 行。
- 每条明细物料启用、数量有效。
- 入库每条明细均有有效单位成本。
- 出库总库存及批次可用库存足够。

### 8.2 二次确认

当满足任一条件时，界面必须在确认过账前显示二次确认弹窗：

- 明细数量汇总超过 `1,000`。
- 入库明细总金额超过等值 `10,000 USD`。

数量汇总用于误操作提醒，即使不同物料的基本单位不同也按保守阈值触发，不作为报表指标。

后端确认命令必须再次计算风险条件；达到阈值而请求未携带用户已确认标记时，拒绝过账，防止绕过前端提示。

### 8.3 原子过账

确认过账必须在一个数据库事务中完成：

1. 锁定并校验草稿单据状态。
2. 校验全部明细和业务类型约束。
3. 根据方向逐行调用库存与批次处理逻辑。
4. 为每条明细写入库存流水。
5. 将单据状态更新为 `confirmed` 并保存确认人、确认时间。
6. 提交事务。
7. 事务成功后写一条整单操作日志。

任意一步失败，事务整体回滚，单据仍保持草稿状态，库存、批次和流水均不得部分生效。

## 9. 数据模型建议

### 9.1 新增单据头表 `manual_stock_movements`

| 字段 | 类型建议 | 说明 |
| --- | --- | --- |
| `id` | `BIGSERIAL` | 主键 |
| `movement_no` | `TEXT UNIQUE NOT NULL` | `FM-*` 单号 |
| `direction` | `TEXT NOT NULL` | `in` / `out` |
| `business_type` | `TEXT NOT NULL` | 第 6 节固定业务类型编码 |
| `warehouse_id` | `BIGINT NOT NULL` | 整单仓库 |
| `movement_date` | `TEXT NOT NULL` | 业务日期 |
| `counterparty_name` | `TEXT` | 借料类往来对象 |
| `remark` | `TEXT` | 整单备注 |
| `status` | `TEXT NOT NULL` | `draft` / `confirmed` |
| `created_by_user_id` | `BIGINT` | 创建人 ID |
| `created_by_name` | `TEXT` | 创建人名称快照 |
| `confirmed_by_user_id` | `BIGINT` | 确认人 ID |
| `confirmed_by_name` | `TEXT` | 确认人名称快照 |
| `confirmed_at` | `TIMESTAMP` | 确认时间 |
| `created_at` / `updated_at` | `TIMESTAMP` | 审计时间 |

数据库或后端必须校验 `direction` 与 `business_type` 的组合合法性，并校验借料类业务类型是否具有往来对象。

### 9.2 新增明细表 `manual_stock_movement_items`

| 字段 | 类型建议 | 说明 |
| --- | --- | --- |
| `id` | `BIGSERIAL` | 主键 |
| `movement_id` | `BIGINT NOT NULL` | 单据头 ID |
| `sort_order` | `INTEGER NOT NULL` | 显示顺序 |
| `material_id` | `BIGINT NOT NULL` | 物料 |
| `quantity` | `DOUBLE PRECISION NOT NULL` | 基本单位数量 |
| `unit_cost_usd` | `BIGINT` | 入库单位成本，USD 最小货币单位 |
| `lot_no` | `TEXT` | 入库批次号 |
| `supplier_batch_no` | `TEXT` | 供应商批次号 |
| `created_at` / `updated_at` | `TIMESTAMP` | 审计时间 |

项目约定不增加数据库外键约束；关联完整性由命令层校验并通过索引保证查询性能。

### 9.3 库存流水兼容策略

现有 `inventory_transactions.transaction_type` 表达系统库存变动分类，正式流程已经依赖该字段做统计。为避免手工记录污染正式业务，首版采用以下策略：

- 批量手工入库流水继续写入 `transaction_type = 'other_in'`。
- 批量手工出库流水继续写入 `transaction_type = 'other_out'`。
- 流水写入 `source_type = 'manual_stock_movement'`、`source_id = manual_stock_movements.id` 和 `related_order_no = movement_no`。
- 查询列表时通过 `source_id` 关联单据头，使用 `business_type` 显示用户选择的类型。

历史上已经生成、但没有单据头记录的单笔自由出入库流水不迁移原因分类，继续显示为「其他入库」或「其他出库」。

## 10. 页面与交互结构

### 10.1 导航与页面名称

- 现有导航「自由出入库」调整为「批量出入库」。
- 现有路由可继续复用 `/manual-stock-movements`，避免无必要的路由迁移。

### 10.2 单据列表页

列表页用于找回草稿和追溯已确认单据，遵循项目业务列表表格骨架组件约定。

| 列 | 说明 |
| --- | --- |
| 单号 | `FM-*` |
| 方向 | 入库 / 出库 |
| 业务类型 | 采购入库、生产领料等 |
| 仓库 | 单据仓库 |
| 往来对象 | 借料类展示对象，其余显示 `-` |
| 变动日期 | 单据业务日期 |
| 明细数 | 物料行数 |
| 状态 | 草稿 / 已确认 |
| 创建人 | 追溯信息 |
| 操作 | 编辑草稿、删除草稿、确认过账、查看详情 |

### 10.3 草稿编辑页

编辑页分为以下区域：

1. 单头信息：方向、业务类型、仓库、日期、往来对象、备注。
2. 快速添加区：物料选择、数量输入、加入明细按钮。
3. 明细表格：行内修改，入库时增加成本和批次字段。
4. 汇总与风险提示：明细行数、数量汇总、入库金额汇总、缺失必填提示。
5. 操作区：保存草稿、确认过账、返回列表。

业务类型选择项随方向切换。切换方向会导致已录入明细的字段语义变化，因此有明细时必须提示用户确认清空明细后再切换。

### 10.4 库存流水页

流水页需增加：

- 手工批量单业务类型的显示。
- 来源单据展示，明确「批量出入库单 FM-*」与正式来源。
- 业务类型筛选及来源筛选，使同名的正式与手工记录可区分查询。

## 11. 后端命令与数据流建议

### 11.1 命令边界

建议新增或重构为以下 IPC 命令：

| 命令 | 用途 |
| --- | --- |
| `get_manual_stock_movements` | 查询批量单列表 |
| `get_manual_stock_movement_detail` | 查询单据头与明细 |
| `save_manual_stock_movement` | 新建或更新草稿，不影响库存 |
| `confirm_manual_stock_movement` | 校验并整单过账 |
| `delete_manual_stock_movement` | 删除草稿 |

现有 `create_manual_stock_movement` 的即时过账路径在新页面替换后不再作为用户入口；是否保留兼容由实施阶段按调用方检索结果处理。

### 11.2 确认过账数据流

入库确认：

1. 为每条明细调用现有增加库存及移动加权平均成本逻辑。
2. 批次追踪物料创建批次记录，批次号为空时自动生成。
3. 为每条明细创建 `other_in` 流水，挂接同一批量单。

出库确认：

1. 为每条明细校验库存数量。
2. 批次追踪物料按 FIFO 查询并锁定可用批次，不允许占用预留数量。
3. 扣减批次和汇总库存。
4. 为每条明细创建 `other_out` 流水，挂接同一批量单。

### 11.3 操作日志

操作日志按整单记录一次，详情至少包含：

- 单号。
- 方向与业务类型。
- 仓库。
- 明细行数。
- 数量汇总。
- 往来对象（仅借料类，避免记录无关信息）。

## 12. 报表与统计调整

| 模块 | 调整 |
| --- | --- |
| 库存收发存、趋势 | 所有已确认手工流水原本即按数量进入统计，继续保留 |
| 智能补货消耗分析 | 在现有正式销售/生产出库之外，纳入业务类型为 `manual_production_out` 的手工出库 |
| 滞销分析中的消耗口径 | 与补货一致，纳入 `manual_production_out` |
| 采购报表 | 不纳入 `manual_purchase_in` |
| 应付账款 | 不纳入 `manual_purchase_in` |
| 生产工单执行与成本 | 不纳入 `manual_production_out` |

## 13. 错误处理

| 场景 | 系统行为 |
| --- | --- |
| 草稿编辑时物料停用 | 禁止新增；已有明细在确认时提示物料不可用 |
| 出库任意一行库存不足 | 拒绝整单过账，错误提示具体物料及可用数量 |
| 出库批次可用量不足 | 拒绝整单过账，错误提示具体物料及批次可用量 |
| 入库缺少成本 | 草稿可保存，确认时拒绝并定位缺失行 |
| 借料类缺少往来对象 | 草稿可保存，确认时拒绝 |
| 其他类型缺少备注 | 草稿可保存，确认时拒绝 |
| 已确认单重复确认或编辑 | 后端拒绝请求 |
| 确认事务中任一操作失败 | 全部回滚，草稿保持可编辑 |

## 14. 验收标准

### 14.1 基础流程

- [ ] 用户可创建入库或出库草稿，单据号格式为 `FM-YYYYMMDD-XXX`。
- [ ] 用户可在一张草稿中编辑最多 100 条明细。
- [ ] 保存草稿后，库存、批次和流水均不发生变化。
- [ ] 用户可重新打开草稿继续编辑，或删除草稿。
- [ ] 确认后单据变为只读，并可查看对应流水。

### 14.2 明细与库存

- [ ] 出库重复物料自动合并数量。
- [ ] 入库相同成本和批次属性的重复物料自动合并数量。
- [ ] 入库相同物料但不同成本或批次可保留多行。
- [ ] 入库确认正确更新移动加权平均成本并生成所需批次。
- [ ] 出库确认正确按 FIFO 扣减可用批次，不扣减已预留数量。
- [ ] 任一明细失败时，整单库存、批次及流水均不生效。

### 14.3 类型与追溯

- [ ] 入库和出库页面仅展示对应方向可选的固定业务类型。
- [ ] 借入、外借及归还类型在确认前要求填写往来对象。
- [ ] 「其他入库」「其他出库」在确认前要求填写备注。
- [ ] 手工「采购入库」「生产领料」在流水中显示自然名称，并清晰显示来源为批量出入库单。
- [ ] 历史单笔 `other_in` / `other_out` 流水仍可正常显示和查询。

### 14.4 统计与风控

- [ ] 手工「生产领料」进入消耗分析、滞销分析相关消耗口径和补货建议。
- [ ] 手工「采购入库」不进入正式采购统计和应付账款。
- [ ] 超出数量或入库金额阈值时，确认过账需要二次确认。
- [ ] 后端在未收到风险确认标记时不得提交高风险单据。

## 16. 已确认决策摘要

| 决策 | 结论 |
| --- | --- |
| 是否值得实施 | 值得，解决高频多物料同时出入库的重复操作 |
| 单据范围 | 同方向、同仓库、同日期、最多 100 行 |
| 生命周期 | 支持草稿，确认后过账，无审批 |
| 明细输入 | 用户逐行选择物料和输入数量，首版不导入 Excel |
| 重复物料 | 出库按物料合并；入库按物料、成本和批次属性合并 |
| 类型建模 | 固定手工业务类型，放在单头 |
| 显示名称 | 使用「采购入库」「生产领料」等自然文案，不显示「无单」字样 |
| 正式流程区分 | 通过来源单据区分手工单与正式业务单 |
| 借料闭环 | 支持借入、借入归还、外借、外借归还，记录往来对象 |
| 采购入库对象 | 不要求填写供应商，不计入正式采购或应付统计 |
| 生产领料统计 | 手工生产领料纳入消耗分析与补货建议 |
