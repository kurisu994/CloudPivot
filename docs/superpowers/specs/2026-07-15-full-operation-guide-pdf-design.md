# CloudPivot IMS 完整操作指南 PDF 设计说明

## 目标

将 `docs/user-manual/all_in_one_manual.md` 制作为适合电脑阅读和 A4 打印的完整操作指南 PDF。成品必须完整保留正文结构、表格、代码块和系统截图，中文清晰，图片不裁切，章节分页稳定。

## 内容范围

- 正文来源：`docs/user-manual/all_in_one_manual.md`。
- 完整保留全部章节与附录，不压缩为精简版。
- 保留 27 处图片引用，对应 `docs/user-manual/images/` 下 26 张原始 PNG。
- 生成时移除 `START OF`、`END OF` 等合并标记。
- 仅在生成流程中修正两处明显乱码：将 `��台创建` 还原为 `当后台创建`，删除独立残片 `�的库存。`；不修改原 Markdown 文件，也不主动改写其他业务内容。

## 页面设计

- 纸张：A4 纵向。
- 方向：均衡阅读版，兼顾屏幕阅读与实体打印。
- 页边距：左右约 18 mm，上下约 16 mm，并为页眉页脚预留安全距离。
- 字体：优先使用系统中文无衬线字体，配置跨平台回退字体。
- 正文：约 10.5 pt，使用舒适行距；标题采用蓝灰色层级。
- 章节：一级章节从新页开始，避免标题落在页尾。
- 图片：使用原始 PNG，保持宽高比，最大宽度不超过正文区域，不裁切、不做有损压缩。
- 表格和代码块：避免横向溢出；长内容允许在合理位置跨页。
- 封面：显示“云枢 CloudPivot IMS 完整操作指南”、版本和更新日期。
- 导航：目录项链接到对应章节。
- 页眉页脚：显示文档名称和连续页码，不暴露本地文件路径。

## 生成方案

采用“语义化 HTML + Headless Chrome 打印”方案：

1. 将合并版 Markdown 转换为语义化 HTML。
2. 注入专用打印 CSS，控制 A4 页面、分页、字体、图片与表格样式。
3. 使用本机 Chromium/Chrome 以打印模式生成 PDF，不为项目新增运行时依赖。
4. 使用 Poppler 将最终 PDF 逐页渲染为 PNG，进行视觉验收。

没有采用 ReportLab，是因为当前环境未安装相关 Python 包，且直接实现复杂 Markdown、中文字体和图片分页的成本更高。没有采用 Pandoc/LaTeX，是因为当前环境缺少对应工具，额外安装会扩大环境改动，并存在中文与 Emoji 字体兼容风险。

## 质量验收

- PDF 可正常打开，页面尺寸为 A4。
- 目录和章节顺序与完整手册一致。
- 26 张图片均被引用，27 处图片位置均能正常显示。
- 中文、英文、越南文、Emoji 不出现黑框或乱码。
- 页面不存在正文截断、图片裁切、表格横向溢出或内容重叠。
- 章节分页、页眉和页码连续且位置稳定。
- 输出文件位于 `output/pdf/cloudpivot-ims-full-operation-guide.pdf`。

- 不重写或删减业务说明。
- 不新增项目依赖或修改前端、Rust 业务代码。
