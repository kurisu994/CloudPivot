## 二、关键问题清单（P0 — 阻塞开发）

> 以下 10 个问题如果不在开发前澄清，将导致返工或数据结构不可逆错误。

### P0-1 ▸ 库存预留生命周期未定义

- **涉及文档**：需求 3.6 / 3.10.3 / 数据库 `inventory_reservations` / UI 2.6
- **问题**：预留何时生成（销售单审核？定制单确认？）、何时消耗（出库确认时？）、何时释放（作废？超期？）的完整规则缺失
- **风险**：定制单预留 → 转销售单 → 销售出库的链路中，极易出现重复扣减或预留泄漏
- **建议**：
  1. 补充「库存预留状态机」文档：`待分配 → 已预留 → 已消耗 / 已释放`
  2. 明确定制单转销售单时，预留的迁移规则（保留 / 重建 / 合并）
  3. 数据库增加 `inventory_reservations.converted_sales_id` 和 `conversion_date` 字段

### P0-2 ▸ 多币种成本核算规则不清

- **涉及文档**：需求 3.4 / 3.5 / 3.7 / 3.8 / 数据库金额字段 / UI 2.7
- **问题**：
  - 汇率固化时点不明确（单据创建时？审核时？入库时？）
  - 移动加权平均法跨币种计算规则缺失（CNY 采购 + USD 采购 → 库存成本如何统一？）
  - 汇兑损益处理方案缺失（应收 CNY 10,000，到账 ¥9,800 时差异如何记录？）
  - 报表 USD 折算使用交易时汇率还是报表期汇率，未说明
- **建议**：补充「多币种成本核算规范」，明确以下 4 个关键决策：
  1. 汇率固化时点 → 建议：单据审核/确认时固化
  2. 库存成本基准币种 → 建议：统一折算为 USD（移动加权）
  3. 报表折算规则 → 建议：使用交易时固化汇率，不回算
  4. 汇兑损益 → 建议：v1.0 仅记录差异，不做自动分录

### P0-3 ▸ 定制单与销售单的状态映射缺失

- **涉及文档**：需求 3.10 / 数据库 `custom_orders` + `sales_orders` / UI 2.9
- **问题**：
  - 定制单（报价中/已确认/生产中/已完成/已取消）与销售单（草稿/已审核/部分出库/已出库）的同步规则未定义
  - 定制单取消后，预留释放、已付定金冲销流程缺失
  - 定制配置修改后，BOM 和预留如何级联更新未说明
- **建议**：
  1. 绘制「定制单 ↔ 销售单」状态映射图
  2. 定义配置变更时的 BOM 重新计算和预留调整规则
  3. 数据库 `custom_orders` 增加 `converted_sales_id` 和 `inventory_reserved_date`

### P0-4 ▸ UI 原型缺失 6 个关键页面

- **涉及文档**：UI 03-ui-prototype.md
- **缺失页面**：
  1. **供应商管理** — 列表 + 新增/编辑表单（含 MST 税号、结算方式、账期）
  2. **客户管理** — 列表 + 新增/编辑表单（含信用额度、客户分类）
  3. **分类管理** — 树状编辑（多级分类、拖拽排序）
  4. **仓库管理** — 列表 + 新增/编辑（仓库类型、负责人）
  5. **采购退货** — 关联原入库单、退货数量、批次反向分配
  6. **销售退货** — 关联原出库单、退货原因、库存恢复确认
- **另外缺失**：销售单详情页（销售出库的前置单据页面未设计）、出入库流水查询页
- **建议**：参照已有的物料管理和采购单页面风格，补齐上述页面的线框图

### P0-5 ▸ 批次追踪的 FIFO 分配与手动改批逻辑未定义

- **涉及文档**：需求 3.6.1 / 3.5.2 / 数据库 `inventory_reservation_lots` / UI 2.13.2
- **问题**：
  - FIFO 的时间基准不明（入库日期？批次号日期？）
  - 人工改批次时，预留同步的触发点和冲突处理规则缺失
  - 数据库无法区分「已 FIFO 分配」和「待人工指定」的预留行
- **建议**：
  1. 明确 FIFO 基准 → 建议按入库日期排序
  2. 补充 `inventory_reservation_lots.allocation_type` 字段（fifo / manual / pending）
  3. 定义改批次后的审计日志和回滚机制

### P0-6 ▸ 仓库粒度模型前后矛盾

- **涉及文档**：需求 3.4.1 / 3.5.1 / 3.6.4 / 数据库约束
- **问题**：采购/销售单声明「v1.0 订单级单仓」，但库存调拨支持多仓操作，定制单跨仓预留逻辑矛盾
- **建议**：
  1. 明确 v1.0 的仓库模型边界：单仓指「一张单据对应一个仓库」，不影响多仓调拨
  2. 数据库增加触发器强制校验明细行仓库与单头一致
  3. 文档中统一描述仓库策略，消除歧义

### P0-7 ▸ 工期严重低估

- **涉及文档**：开发计划 04-development-plan.md
- **问题**：
  - 官方 22–24 周，按任务清单自下而上估算实际需 **28–32 周**（单人）
  - 阶段三（核心业务）4 条业务线完全并行，单人开发不可行
  - 各阶段缓冲时间仅 1–5 天，远不足以应对实际延期
- **建议**：

  | 阶段 | 原估计 | 建议修正 | 说明 |
  |------|:------:|:-------:|------|
  | 阶段一：脚手架 | 4 周 | 5 周 | 增加 Tauri POC 验证 + 缓冲 |
  | 阶段二：基础数据 | 3.5 周 | 4.5 周 | 任务量实际 14+ 工作日 |
  | 阶段三：核心业务 | 7.5 周 | 9 周 | 改为顺序开发：采购→销售→库存→定制 |
  | 阶段四：财务报表 | 3.5 周 | 4 周 | 增加报表性能测试 |
  | 阶段五：发布翻译 | 4.5 周 | 5.5 周 | 增加翻译校对和跨平台测试 |
  | **合计** | **22 周** | **28 周** | 单人全栈开发 |

### P0-8 ▸ 出库成本固化规则不清

- **涉及文档**：需求 3.5.2 / 3.8.2 / 数据库 `outbound_order_items`
- **问题**：销售单可分批出库，每批出库成本不同；订单级毛利分析如何聚合未说明
- **建议**：明确「每次出库独立固化成本快照，订单毛利 = Σ(各批出库金额 - 各批成本)」

### P0-9 ▸ 销售/采购退货的成本反向调整未设计

- **涉及文档**：数据库 `sales_return_items` / `purchase_return_items`
- **问题**：退货单缺少 `cost_unit_price` / `cost_amount` 字段，无法反向调整库存成本
- **建议**：
  1. `sales_return_items` 增加 `cost_unit_price`、`cost_amount`、`cost_reversal_amount`
  2. `purchase_return_items` 增加 `receipt_unit_cost`、`cost_reversal_note`

### P0-10 ▸ 应付/应收的汇率折算数据不完整

- **涉及文档**：数据库 `inbound_orders` / `payables` / `receivables`
- **问题**：`inbound_orders` 缺少 `total_amount_base`（基准币折算金额），分批入库时每批汇率可能不同，无法追踪「按哪个汇率生成的应付」
- **建议**：
  1. `inbound_orders` 增加 `total_amount_base` 字段
  2. `payables` 增加 `exchange_rate_snapshot` 和 `source_inbound_id`

| 11 | **补货建议参数来源不清**：日均消耗的时间窗口、安全天数的配置位置未说明 | 需求/UI | 补充补货参数与物料表/系统配置的映射关系 |
| 12 | **财务账期计算规则不清**：到期日的计算基准和超期判断逻辑未说明 | UI | 补充到期日 = 入库日 + 账期天数；部分付款不重算到期日 |
| 13 | **术语不统一**：「物料」vs「产品」vs「成品物料」混用 | 需求 | 统一术语表：物料（总称）→ 原材料 / 半成品 / 成品（类型标签） |
| 14 | **关联单据的操作限制未明确**：已入库的采购单能否修改价格？已出库的销售单能否删除？ | 需求/UI | 补充「单据状态与操作权限矩阵」 |
| 15 | **阶段三测试时间严重不足**：各阶段仅预留 1 天测试 | 开发计划 | 每个阶段至少预留 15–20% 时间用于测试和联调 |
