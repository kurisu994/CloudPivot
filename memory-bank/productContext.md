# Product Context — 用户流、业务约束与交互逻辑

## 行业背景

越南家具生产行业的核心特点：

- **原材料种类繁多**：木材（实木、板材）、五金配件、油漆涂料、布艺皮革、玻璃、包装材料
- **半成品/成品两级库存**：毛坯件、组装件等半成品形态
- **BOM 刚需**：一件成品家具由数十种原材料组成，需精确管理用量和损耗率
- **批次追溯**：木材含水率、板材甲醛等级需按批次追溯
- **定制化生产**：非标定制单是日常业务
- **跨国供应链**：供应商来自越南本地、中国、马来西亚等地

## 越南工厂特殊约束

| 维度 | 约束 |
|------|------|
| 货币 | VND（日常）/ CNY（中国供应商）/ USD（出口），金额统一以最小货币单位 INTEGER 存储 |
| 语言 | 中文 / 越南语 / 英文 三语切换，工厂管理层可能是中方/越方人员 |
| 税务 | MST（Mã số thuế）税号，增值税率与中国不同 |
| 地址 | 省 → 市/县 → 坊/社 → 街道 |
| 电话 | 国际区号 +84，手机号 10 位 |
| 时区 | UTC+7（Ho Chi Minh），数据库连接统一设置 |

## 核心用户流

### 采购流程
```
供应商管理 → 创建采购单 → 审核 → 入库确认（批次+费用分摊）→ 应付账款 → 付款登记
                                  ↘ 退货确认（成本回调）→ 应付冲减
```

### 销售流程
```
客户管理 → 创建销售单 → 审核 → 出库确认（FIFO+双成本快照）→ 应收账款 → 收款登记
                                ↘ 退货确认（成本回调+应收冲减）
```

### 定制 & 生产流程
```
定制单 → 配置BOM → 预留库存 → 转生产工单 → 领料 → 生产 → 完工入库 → 转销售单
```

### 库存管理流程
```
库存查询（四维筛选+预警） → 盘点（实盘录入+审核） → 调拨（双仓选择） → 自由出入库（草稿+过账）
                          ↗ 出入库流水（多维筛选+来源区分）
```

### 智能补货流程
```
策略配置 → 系统计算建议 → 审批/忽略 → 一键生成采购单
```

## 交互逻辑要点

- **桌面端优先**：Tauri 原生窗口，默认最大化，最小 1024×768
- **列表页统一骨架**：`BusinessListTableShell` + 表头吸顶 + 首列 sticky + 分页吸底
- **编辑/详情用 Dialog**：不用 Drawer/Sheet，统一 Dialog 弹窗
- **日期选择**：带弹出日历的 DatePicker，列表页用日期范围选择器
- **紧凑视图**：外观设置可切换紧凑列表视图，全局生效
- **权限驱动 UI**：菜单和按钮按角色权限自动隐藏
- **Toast 提示**：统一使用 Sonner，不手写 Toast
- **橡皮筋回弹已禁用**：接近原生桌面体验
- **自动更新**：启动后自动检查，关于弹窗提供手动检查入口

## 数据安全约束

- SQL 查询全部参数化，禁止字符串拼接
- 库存操作使用 `FOR UPDATE` 行锁
- IPC 写命令统一 `require_auth` 守卫
- 用户管理类命令需 admin 角色
- 认证会话文件 Unix 0600 权限
- 日志不记录密码、证件号等敏感信息
