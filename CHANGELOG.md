## [0.2.5] — 2026-06-10

### 新增

- **自由出入库单据打印**：已过账的自由出入库单可在列表行内直接打印；浏览器打印对话框出现后可选系统已安装的打印机出纸，afterprint 后自动回列表。每次打印自动记录审计日志便于事后追溯。
- **打印模板可视化设计器**：管理员可在「系统设置 → 打印设置 → 编辑模板」里调整列的可见性、顺序、宽度（6 档）和对齐（左/中/右），实时预览所见即所得；支持「重置为默认」一键回滚。
- **打印设置页接通持久化**：「显示项」三个开关（Logo / 公司信息 / 日期与页码）真实保存到后台并影响打印结果；模板列表显示哪些已被自定义；实时预览面板用示例数据渲染所选模板。
- **打印模板系统底座**：通用模板/数据源/渲染层抽象，自由出入库做首发模板；预留采购入库、销售出库、盘点、调拨、生产工单等模板的接入点。

### 变更

- **打印纸张默认为 14×22cm 连续三联纸**：默认纸张定为 14×22cm 三联连续纸，配合等宽字体（中文宋体 / 越南文 Noto Sans Mono CJK）确保数字不错位；适配针式打印机的击打复写工艺。

## [0.2.4] — 2026-06-08

### 新增

- **数据备份操作进度提示**：创建备份、恢复备份、删除备份操作期间显示持久化的加载提示，操作完成后自动替换为结果反馈，支持中越英三语。

### 变更

- **自由出入库默认选中原材料仓**：新建自由出入库单时，仓库字段自动选中原材料仓，减少重复操作。
- **移除顶部快捷还原按钮**：数据管理页面移除顶部的"还原备份"快捷按钮，避免误操作；用户可通过备份历史列表中每行的"恢复"按钮选择具体备份进行还原。
- **补货策略配置弹窗体验改造**：弹窗内搜索、表头、分页栏改为常驻可见，仅表格行参与滚动；弹窗整体放宽，列较多时表格不再被挤压。
- **用户管理新建/编辑弹窗体验优化**：必填项加红色星号标识；保存时缺项不再弹顶部提示，改在对应字段下方就近提示，重新输入或选择后提示自动消失；新建用户默认角色调整为「查看者」。

### 修复

- **数据备份恢复部分情况下中途失败**：修复备份恢复在特定数据下中途失败的问题；恢复完成后自动校正内部计数，确保后续新增数据不会冲突。
- **仪表盘近 30 天数据凌晨偏差**：修复仪表盘销售排行、采购趋势、销售趋势在凌晨时段近 30 天日期范围整体偏移一天的问题。

## [0.2.3] — 2026-06-06

### 新增

- **自由出入库库存不足预检与精确提示**：在自由出入库单确认过账（出库类业务）时，新增库存预检逻辑。若有物料库存不足，系统将整单原子回滚，并以表格弹窗精确展示所有不足物料的编码、名称、需求数量、当前可用量和差额，便于用户快速定位和调整。
- **零库存行与差额突出高亮**：库存不足提示弹窗中，对于可用库存为零的物料行进行底色高亮，且差额数值以红色加粗显示，强化视觉警示。
- **列表页库存不足预检弹窗**：在自由出入库列表页面对草稿单进行确认过账时，同样支持库存不足预检结果的表格弹窗展示，与编辑页体验保持一致。

### 变更

- **明细表格重构为标准组件**：将自由出入库编辑页面的明细表格从原生 HTML 表格重构为标准的 UI 库表格组件，提升界面整体美观度和一致性，且所有表头文案均已接入多语言。
- **确认过账流程静默保存草稿**：在编辑页面点击"确认过账"时，系统内部会先保存草稿，但不再弹出"草稿已保存"的提示，避免用户感知到不必要的中间状态。
- **列表页确认失败支持直接重试**：在列表页面对草稿单进行确认过账时，若因非风控/非库存原因失败，确认对话框不再自动关闭，用户可以直接再次点击确认按钮重试。

### 修复

- **自由出入库对话框 HTML 嵌套错误**：修复风控二次确认对话框和普通确认对话框中，`DialogDescription` 组件内部嵌套了块级元素（`<div>`、`<p>`）导致的浏览器控制台报错（`<p> cannot contain a nested <div>`）。

## [0.2.2] — 2026-06-04

### 变更

- **全站业务表格布局与滚动体验优化**：将表格区域独立滚动、表头固定、分页栏始终吸底可见的紧凑布局推广至全站所有业务列表与明细表格。即便在数据行数极多时，表头与分页控制也无需向下滑动即可操作，消除了内容撑满整个屏幕且滚动条无法触及的交互缺陷。
- **每页显示条数预设与默认值调整**：丰富了分页器每页显示条数的预设选项，并调大规范了全站表格的默认单页加载行数，使大批次数据的展示更加连贯。
- **单位管理列表统一为全站业务表格风格**：移除页面顶部与卡片内的冗余标题，新增按钮固定在表格右上角；表头吸顶、第一列名称居中，与其他业务列表观感一致。
- **禁用页面滚动到边界时的橡皮筋回弹**：滚动到顶部或底部时不再出现整页抖动/回弹，更接近原生桌面应用体验。

### 修复

- **库存报表切换每页条数后数据不刷新**：修复切换每页显示条数后列表数据未重新加载的问题，同时自动回到第一页。
- **库存查询 / 出入库流水 / 库存调拨 / BOM 管理分页栏未真正吸底**：原分页栏嵌在表格滚动容器内部，数据较多时会随内容一起被推到屏幕外；现已拆到滚动容器之外，分页栏始终固定在页面底部可见。


## [0.2.1] — 2026-06-02

### 新增

- **多账号与角色权限体系**：从单管理员模式升级为多账号管理，内置「管理员 / 操作员 / 查看者」三个角色。管理员可在「系统设置 → 用户管理」创建账号、分配角色、重置密码、禁用/启用和解锁；新账号首次登录强制改密。导航菜单和各业务页的操作按钮按当前用户权限自动隐藏。

### 变更

- **日期选择体验升级**：所有日期字段（采购单、销售单、定制单、生产工单、出入库执行、盘点、调拨、应付/应收登记、BOM 生效日、供应商物料有效期等）改为带弹出日历的日期选择器，可在日历中翻月与选日；列表与报表页的「起始日期 / 截止日期」两个输入合并为统一的日期范围选择器，一次点击框选完整区间。日历表头与月份名按界面语言显示（中/越/英）。
- **角色权限范围收紧**：查看者不再可以进入自由出入库、库存盘点、智能补货与通用设置；操作员的物料管理收为仅查看，分类管理、单位管理、报表中心、通用设置一并下放回管理员；外观设置对所有角色保持可编辑。物料管理的导入与导出收为独立权限，查看者不再看到这两个按钮。
- **紧凑列表视图真正生效**：外观设置中的「紧凑列表视图」开关此前点击后无变化，本次接入全站表格组件，开启后整体行高与单元格内距明显减小，可在不调整列宽的前提下显示更多行。

## [0.1.9] — 2026-05-27

### 新增

- **退出登录入口**：顶栏右上角用户区改为下拉菜单，展示当前登录用户（昵称与角色）并提供「退出登录」操作，清除会话并跳转登录页。

- **出入库流水来源区分**：流水页新增「来源」列及筛选，区分自由出入库单与正式单据，手工单不污染采购报表、供应商排行、应付账款、工单成本等正式统计口径。
- **盘点创建与录入流程优化**：盘点单创建时预填系统库存为实盘数量，操作员只需修改差异行；创建弹窗增加加载反馈，避免重复提交。

- **库存报表暂时隐藏金额列**：收发存 / 库龄表的金额列、顶部金额 KPI 卡片与导出 Excel 金额列暂时隐藏，由开关统一控制。
- **全局表格分页器统一升级**：统一分页器视觉与默认每页条数，移除冗余「回到首页」按钮，精简操作日志底部分页统计文字。
- **系统设置页与顶栏微调**：系统设置页去除两侧最大宽度限制、内容铺满整屏；移除顶栏未启用的通知铃铛占位按钮。

- **退出登录未记录操作日志**：此前退出动作在操作日志中缺失，现已补齐 logout 日志记录。
- **数据库时间戳时区漂移**：连接池新增时区设置钩子，统一使用 UTC+7，确保写入时区稳定且与宿主机无关。
- **盘点明细创建耗时**：物料级、批次级快照由逐行写入改为参数化集合插入，创建耗时不再随明细条数增加数据库往返。
- **自由出入库出库流水方向错误**：批量出库确认时库存流水数量未取负，导致出库记录显示为入库方向，已修正。
- **多处缺失 i18n 文案补齐**：补齐自由出入库列表合计、单位 / 页码 / 上一页 / 下一页、客户删除失败提示等缺失键。

## [0.1.8] — 2026-05-25

### 变更

- **主窗口默认最大化**：应用启动时窗口自动最大化填满屏幕工作区，保留标题栏并可还原；macOS/Windows/Linux 三平台行为一致。
- **菜单阶段性开放**：侧边栏当前仅开放首页看板、物料/分类/单位管理、库存查询/自由出入库/出入库流水/库存盘点、智能补货、操作日志、外观设置；其余模块暂时隐藏，后续分批开放。

### 修复

- **Critical/High 级别安全与并发缺陷**：
  - SQL 注入防护：操作日志、盘点单、单号生成等模块改用参数化查询，消除字符串拼接风险
  - PostgreSQL 方言兼容：统一使用 `GREATEST` 替代 `MAX`
  - 并发覆盖保护：库存查询添加行锁，批次预留增加数量约束校验
  - IPC 鉴权增强：写命令统一添加认证守卫
  - 密码修改安全：后端增加旧密码校验，前端改密页同步添加旧密码输入框
  - 迁移原子性：每条迁移在独立事务内执行
  - BOM 删除保护：修正被引用字段判断条件
  - 备份恢复安全：改为事务内执行并添加鉴权校验
- **构建环境区分**：仅在 Tauri 构建生命周期启用静态导出，开发模式保持 Next.js 服务器。
- **i18n 翻译补全**：操作日志页面补全三语下缺失的加载与空数据翻译键。
- **WebView 打印权限缺失**：补全打印权限，使单据打印能正常唤起打印对话框。
- **应付/应收 KPI 货币单位错误**：KPI 卡片金额硬编码按 VND 格式化，统一改为基准币种 USD。
- **多处硬编码中文文本**：清理 BOM 编辑/反查、定制单详情、物料表单、企业信息与打印设置页中的中文硬编码，改为国际化调用。
- **物料分类筛选选父分类无结果**：改为按子树查询，选父分类可列出其全部子孙分类下的物料。
- **报表与库存查询的 PostgreSQL 类型不匹配**：统一补全类型转换，修复日期相减和文本列比较导致的解码失败。

- **分类管理删除无效果**：改用 Dialog 组件实现确认弹窗，修复 Tauri WebView 下原生确认框被静默拦截的问题。

- **输入框首字母自动大写**：全局禁用 autocapitalize、autocorrect 和 spellcheck。
- **分类管理新增/更新无法挂到父级**：统一字段为 camelCase，修复新建分类总是落到顶级节点的问题。

### 重构

- **全站替换原生确认框为统一确认弹窗组件**：新增 ConfirmDialog 组件，覆盖 18 个业务页面，支持危险操作样式。

## [0.1.5] — 2026-05-13

### 重构

- **数据库迁移 SQLite → PostgreSQL**：将嵌入式 SQLite 替换为远程 PostgreSQL，支持多终端共享数据和集中式数据安全。
- **采购/销售共享抽象**：提取两个模块中重复模式，消除约 516 行重复代码。

### 新增

- **统一 Loading 状态设计系统**：提供 6 种场景化 Loading 组件，统一全应用加载态视觉风格。
- **Dashboard 快捷操作按钮功能**：4 个快捷按钮添加路由导航，点击直接跳转对应页面。
